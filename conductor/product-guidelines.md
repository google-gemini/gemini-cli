# Product Guidelines

## Prose Style

- **Professional & Direct:** Use a professional, direct, and concise tone
  suitable for a CLI environment.
- **Action-Oriented:** Focus on what the user can do and how the tool assists
  them.
- **Minimal Filler:** Avoid conversational filler, apologies, and unnecessary
  preambles.
- **Clarity First:** Prioritize technical rationale and clear intent in all
  communications.

## Branding & Identity

- **Name:** Always refer to the product as "Gemini CLI".
- **Visuals:** Use high-quality, modern aesthetics for terminal rendering.
- **Consistency:** Maintain a consistent look and feel across all CLI components
  and documentation.

## User Experience (UX) Principles

- **Terminal-First:** Optimize all interactions for the command line.
- **Feedback:** Provide immediate and clear feedback for all user actions and
  tool executions.
- **Safety:** Explicitly warn users before executing potentially destructive or
  sensitive commands.
- **Efficiency:** Minimize the number of steps required to achieve a goal.
- **Accessibility:** Ensure the CLI is accessible to users with different
  terminal configurations and needs.

## Design Guidelines

- **Modern UI:** Use stylized shapes, gradients, and platform-appropriate design
  elements (via Ink).
- **Responsive Layout:** Ensure the UI adapts gracefully to different terminal
  window sizes.
- **Interactivity:** Provide rich, interactive feedback to make the application
  feel "alive."
