# Product Guidelines

## Prose Style

- **Tone:** Professional, direct, and concise. Avoid conversational filler.
- **Voice:** Active and technical. Speak directly to the developer\u0027s
  intent.
- **Formatting:** Use clear, consistent Markdown. Prioritize readability in a
  terminal environment.

## Branding \u0026 Identity

- **Product Name:** Always use "Gemini CLI".
- **Visual Identity:** Leverage ANSI colors and rich terminal formatting (via
  Ink) to provide high-signal feedback. Maintain a "modern terminal" aesthetic.

## UX \u0026 Design Principles

- **Terminal-First:** Every feature must be optimized for a keyboard-driven
  workflow.
- **Low Latency:** Prioritize perceived performance and immediate feedback for
  all user actions.
- **Predictability:** Commands and interactions should follow established CLI
  conventions (e.g., standard flags, clear exit codes).
- **Safety:** Explicitly warn before performing destructive or high-impact
  operations.
- **Discoverability:** Provide robust help systems and intuitive slash commands.
