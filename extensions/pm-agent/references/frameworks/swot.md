# SWOT Analysis Framework

## Overview

SWOT (Strengths, Weaknesses, Opportunities, Threats) is a strategic planning
framework for evaluating a project's competitive position.

## How to Apply to OSS Projects

### Strengths (Internal, Positive)

- Active contributor community
- Strong documentation
- Fast release cadence
- Broad platform support
- Clear architecture
- Good test coverage

### Weaknesses (Internal, Negative)

- Growing backlog of unresolved issues
- Declining contributor activity
- Missing features vs. competitors
- Complex onboarding process
- Technical debt accumulation
- Poor error messages or UX

### Opportunities (External, Positive)

- Growing market demand for the tool category
- New platform or integration possibilities
- Community interest in specific features
- Enterprise adoption potential
- Partnership or ecosystem opportunities
- Emerging technologies to leverage

### Threats (External, Negative)

- Competing projects gaining traction
- Key maintainer burnout risk
- Breaking changes in dependencies
- Shifting industry standards
- Funding or sustainability concerns
- Security vulnerability exposure

## Output Format

Present as a 2x2 matrix:

|              | Positive      | Negative   |
| ------------ | ------------- | ---------- |
| **Internal** | Strengths     | Weaknesses |
| **External** | Opportunities | Threats    |

Follow with strategic recommendations that leverage strengths to capture
opportunities and mitigate threats.
