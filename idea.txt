    The Bridge: HATS MCP Server Instead of the CLI calling hats via raw shell commands, build a specialized MCP server in Python. Implementation: Use the Python MCP SDK (or FastMCP) to wrap your hats-framework functions as "Tools". Benefit: Gemini will receive structured JSON findings from HATS instead of messy terminal text. This allows it to "understand" open ports or vulnerabilities rather than just "reading" them.
    The Package: Gemini-Cyber Extension Bundle your customizations into an official Gemini Extension structure within your fork. gemini-extension.json: Define your HATS MCP server here. This ensures that whenever the cyber-cli starts, the HATS tools are automatically active. GEMINI.md (System Context): Hardcode your "Mission Statement."     Offensive "Agent Skills" Use the Agent Skills feature to keep the model fast. Create a skills/recon directory for active scanning instructions. Create a skills/exploit directory for vulnerability verification logic. Logic: These specialized instructions only load when Gemini identifies a relevant task, saving token space and improving response speed.
