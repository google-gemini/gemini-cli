# Heap Snapshot Analyzer

Skill for analyzing Node.js heap snapshots to diagnose memory issues.

## Capabilities

- **Object Cluster Analysis**: Find groups of related objects consuming memory
- **Growth Pattern Detection**: Identify unexpectedly large collections
- **Detached DOM Detection**: Spot detached HTML elements (browser context)
- **Retainer Chain Analysis**: Trace why objects are kept alive
- **Summary Generation**: Human-readable reports with actionable insights

## Usage

```bash
# Analyze a single heap snapshot
@heap-snapshot-analyzer analyze <path/to/snapshot.heapsnapshot>

# Compare two snapshots (before/after)
@heap-snapshot-analyzer compare <before.heapsnapshot> <after.heapsnapshot>

# Find specific object types
@heap-snapshot-analyzer find --type "Array" --min-count 1000 <snapshot.heapsnapshot>

# Generate Perfetto trace
@heap-snapshot-analyzer to-perfetto --input <snapshot> --output <trace.json>

# Export retained size analysis
@heap-snapshot-analyzer retained-size <snapshot.heapsnapshot>
```

## Analysis Dimensions

### 1. Type Distribution
```
┌─────────────────┬─────────┬──────────┐
│ Type            │ Count   │ Size     │
├─────────────────┼─────────┼──────────┤
│ (array)         │ 15,432  │ 2.1 MB   │
│ Object          │ 8,120   │ 1.8 MB   │
│ string          │ 45,230  │ 3.4 MB   │
│ DetachedHTMLDiv │ 1,200   │ 240 KB   │ ← suspicious
└─────────────────┴─────────┴──────────┘
```

### 2. Object Clusters
- Arrays with >1000 elements
- Objects with >100 properties
- Deeply nested structures (>10 levels)
- Circular references

### 3. Suspicious Patterns
- **Detached DOM**: Elements no longer in document but retained
- **Event Listeners**: Accumulating without removal
- **Closures**: Holding references to large scopes
- **Timers**: setInterval/setTimeout without cleanup
- **Cached Data**: Unbounded caches

### 4. Growth Indicators
- Objects growing between snapshots
- Increasing retained sizes
- New object types appearing

## Example Workflows

### Detecting Memory Leak
```
User: "My app memory keeps growing"

1. Get initial snapshot: @heap-snapshot-analyzer analyze heap-1.heapsnapshot
2. Run workload for 5 minutes
3. Get second snapshot: @heap-snapshot-analyzer analyze heap-2.heapsnapshot
4. Compare: @heap-snapshot-analyzer compare heap-1.heapsnapshot heap-2.heapsnapshot
5. Generate Perfetto trace for visual inspection
```

### Finding Large Object Clusters
```
User: "Which objects are using the most memory?"

@heap-snapshot-analyzer top-objects --limit 20 heap.heapsnapshot
```

## Technical Details

Heap snapshots are generated by:
- Chrome DevTools: Memory tab → Take snapshot
- Node.js: `v8.writeHeapSnapshot()` or `--heapsnapshot-near-heap-limit`
- Programmatic: `require('v8').writeHeapSnapshot()`

## Implementation

Uses shared utilities from `skills/shared/perfetto.ts`:
- `HeapSnapshot` interface for parsing
- `analyzeHeapSnapshot()` for pattern detection
- `diffHeapSnapshots()` for comparison
- `PerfettoTraceBuilder` for trace generation
