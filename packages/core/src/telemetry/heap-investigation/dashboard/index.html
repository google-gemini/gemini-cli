How to generate a CPU profile
Run Gemini CLI with the CPU profiler enabled:
node --cpu-prof --cpu-prof-dir=/tmp gemini-cli
Or via the inspector in DevTools:
node --inspect gemini-cli
# Open chrome://inspect → DevTools → Profiler tab → Record
The .cpuprofile file will be analyzed here to identify hot functions, jank, and performance regressions in the Gemini CLI agent loop.

No trace loaded. Run a heap investigation to generate one.
How to read the trace
1. Open ui.perfetto.dev
2. Click Open trace file and select the .perfetto.json
3. Look for agent_turn_N slices — each represents one LLM call
4. Flow arrows (orange) link memory spikes to the causative agent turn
5. Click a threshold_exceeded event to see which tool call caused the spike
Tip: Use W/S to zoom in/out, A/D to pan left/right in Perfetto UI
